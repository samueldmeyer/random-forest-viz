A Path to Random Forest

A visual explanation of the random forest algorithm


Random forest is a crowdsourcing machine learning algorithm that builds on decision trees, another machine learning algorithm.

Decision tree predicts the value of a target variable, e.g. classification of a data point into one of the given categories, based on its input attributes. To train a decision tree, we give it many examples, and the model tries to find the most informative attributes that help us predict a new instance. In the classification example, the algorithm looks for the most informative attributes that differentiate different categories from each other.

The final structure of a trained decision tree looks like a flow-chart. It asks a single question about the test item, moves onto the next question based on the answer, and then asks more questions until it reaches a conclusion.

Check one out below! Hover over the highlighted circles to see more details about the different components of a decision tree. If you want to know more, check out R2D3's Intro to Decision Trees.

The problem with decision trees is that they either have blind spots based on the data that was used to create them in the first place. Random forest solves this by making a bunch of trees (a forest, also known as an ensemble) and combining the results together.

When we use one predictor, even a pretty good one, sometimes it makes mistakes.
A different predictor might give pretty good but slightly different predictions.
What if we can combine the power of the individual predictors to minimize our mistakes?
This brings us back to the idea of random forest, where we combine the power of individual predictors - decision trees in this case - and achieve overall better result. To illustrate how it works, we here introduce an exciting problem:

Can I eat this mushroom?




We have a dataset of 8000 mushrooms with 22 attributes, and we want to predict if a mushroom is edible or poisonous based on their attributes.
If all trees were trained on exactly the same data and attributes, the trees would all be the same, so they would all predict the same answer, which wouldnâ€™t be any better than just having one. Instead, each tree is given a subset of the data and trained on just that subset. As a result, each tree is a specialist in using some subset of the data.

As each tree is trained on different data, they grow different decision paths. Click each tree to see their individual judging criteria for if a mushroom is edible or poisonous.
